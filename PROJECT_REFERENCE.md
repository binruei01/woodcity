# 嘉義木都導覽小學堂~專案製作參考指南

這份文件紀錄了建立此互動式文化導覽網站的必要元素、提示詞與技術架構，供未來製作類似專案時參考。

---

## 1. 核心提示詞 (AI Prompts)

### 系統指令 (System Instruction)
這是 AI 行為的核心。它定義了導覽員的專業範圍、說話口吻以及當遇到無關問題時的應對方案。

```text
你是一位專業的嘉義木都文化導覽員。你的任務是根據以下知識庫回答使用者的問題。
請使用親切、適合國小高年級學生的口吻回答。

知識庫內容：
1. 嘉義木都：嘉義緊鄰阿里山，擁有高品質的紅檜與扁柏。日治時期建了森林鐵路，並在嘉義市區設立大型製材所。
2. 東市場：屋頂使用不用釘子的「卡榫」技術，運用檜木與杉木搭建。
3. 阿里山森林鐵路：具備世界級普世價值的「遺產潛力點」。
4. 檜木：阿里山產紅檜和扁柏，木質堅硬且有特殊香氣。
5. 老屋：嘉義市有很多「舊屋力」計畫修復的木構老房子。
6. 卡榫：一種不用釘子的智慧技術，利用木頭凹凸結構嵌合。

回答限制：
- 若問題無關，請友善引導回核心話題。
- 使用繁體中文，適合國小生閱讀。
```

## 2. 網站架構 (Architecture)

### 前端技術棧 (Tech Stack)
- **框架**：React 18 + TypeScript (強型別確保程式穩定)
- **構建工具**：Vite (快速、具備現代化開發環境)
- **樣式**：Tailwind CSS (使用 Utility-first 類名快速切換響應式佈局)
- **動畫**：Framer Motion (實現流暢的進場與卡片過場效果)
- **圖示**：Lucide React (現代、乾淨的向量圖示)

### 功能模組
1. **導覽列**：包含專案名稱與「系統狀態」診斷鈕。
2. **英雄區塊 (Hero)**：大標題與副標題，使用深色背景營造木質莊重感。
3. **核心知識卡片**：利用 Grid 佈局呈現重點知識。
4. **互動問答區**：包含搜尋框與 AI 回答區。
5. **歷史紀錄**：自動儲存當次對話。
6. **頁尾**：標註製作團隊與版權資訊。

## 3. 手機端優化策略 (Mobile Best Practices)

- **避免縮放**：使用特定的 Meta Viewport 標籤防止手機輸入時強制縮放。
- **鍵盤交互**：使用 `inputMode="search"` 並將輸入框放在 `form` 中，讓手機鍵盤顯示「搜尋」鈕。
- **視覺大小**：按鈕與觸控點至少保持 44px 高度，字體不小於 16px (避免 iOS 自動縮放)。
- **Polyfill**：在 `main.tsx` 加入 `process.env` 墊片，防止 mobile 瀏覽器噴錯。

---

## 4. API 金鑰與安全 (Security)

- **金鑰處理**：API Key 應儲存在 Secrets 變數中，不要直接寫在程式碼裡。
- **系統診斷**：建立一個「系統狀態」面板，幫助使用者在手機端檢查連線環境。

---

## 5. 設計風格建議 (Design System)

- **配色 (Palette)**：
  - 主基調：米白色 (`#fffaf0`) - 溫暖、清爽。
  - 文字色：深咖啡色 (`#5d4037`) - 代表木頭質地。
  - 重點色：森林綠 (`#2e7d32`) - 代表阿里山與環保。
  - 警告色：琥珀色 (`#ffc107`) - 代表提示與重點。
- **字體**：偏向 Sans-serif (無襯線體)，確保在各螢幕下皆清晰易讀。
